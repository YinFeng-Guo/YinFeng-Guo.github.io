Heroku使用注意点

如何让开发更快速，如果让更改更容易，哪些是heroku比较特殊的地方，总结在项目中遇到过的问题

使用环境变量，不要hard code

使用heroku config命令可以查看一些heroku里设置的环境变量

我们在code里可以利用这些环境变量，而不要硬编码
比如我们在项目里使用了postgres，
如果我们硬写进去，我们可能会写出下面的代码：
public final static String host = “host.eu-west-1.compute.amazonaws.com”;
public final static String database = “database”;
public final static String username = “username”;
public final static String password = “password”;
public final static String port = “port”;
public static String dbUrl = “jdbc:postgresql://“ + DBUtil.host + ‘:’ + DBUtil.port + “/“ + DBUtil.database + “?username=” + DBUtil.username +
“&password=” + DBUtil.password + “&ssl=true&sslfactory=org.postgresql.ssl.NonValidatingFactory”;
